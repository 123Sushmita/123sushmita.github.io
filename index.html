Project Workflow: Credit Card Fraud Detection

Step 1: Data Understanding

Explored the dataset to understand features like transaction amount, transaction hour, merchant category, and fraud indicator. Identified class imbalance where fraud cases are rare.

Step 2: Data Analysis using Python

Used Python (Pandas, Matplotlib, Seaborn) to clean data, analyze distributions, and identify patterns. Found that fraud transactions tend to have higher amounts and higher transaction frequency.

Step 3: SQL-Based Fraud Scoring

Built SQL queries using CASE statements to assign fraud risk scores based on multiple factors such as transaction amount, foreign transactions, location mismatch, and device trust score.

Step 4: Visualization in Tableau

Created dashboards to visualize fraud patterns including fraud vs non-fraud count, fraud by amount, fraud by hour, and risk indicators.

Step 5: Key Insights

  • Fraud transactions have higher amounts
  • Foreign transactions show higher fraud rates
  • High transaction velocity indicates suspicious behavior
  • Combining features improves fraud detection accuracy
